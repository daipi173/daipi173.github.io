JavaScript之易错点

1. 易错点一

undefined与null相等，但不恒等（===）;
null的类型是对象，undified的类型是undified;

2. 易错点二

为什么0.1 + 0.2 不等于0.3。因为计算机不能精确表示0.1， 0.2这样的浮点数，计算时使用的是带有舍入误差的数。
